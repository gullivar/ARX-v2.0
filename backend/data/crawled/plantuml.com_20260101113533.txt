Relies on an external program; Smetana . (Use !pragma layout smetana in the file or -Playout=smetana on the command-line.) A port of Graphviz to Java. Tends to make slightly straighter arrows; VizJs uses JavaScript. (Use -graphvizdot vizjs on the command line.) Tends to space out nodes more, resulting in a bigger diagram; ELK (Eclipse Layout Kernel) . (Use !pragma layout elk in the file or -Playout=elk on the command-line.) Supports only orthogonal layout, and doesn't cover all features. Sequence diagrams have 2 engines: Puma is the older and still default engine; Teoz (Use !pragma teoz true in the file or -Pteoz=true on the command-line) is a new engine that has some unique features: anchors (named arrows/messages), duration, nested boxes... Achieving good layout with PlantUML is sometimes non-trivial. See PlantUML GraphViz Layout by Mark George at University of Otago for more advice.
